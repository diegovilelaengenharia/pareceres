## 3. Requisitos Técnicos

### 3.1 Geração de Texto Final (Zero Placeholders)
A IA deve processar os dados extraídos dos PDFs e as aprovações do engenheiro para gerar blocos de texto completos.
- **Proibido:** Uso de colchetes como `[REQUERENTE]` ou instruções como `[PREENCHER]`.
- **Obrigatório:** Substituição dinâmica de todos os valores (nomes, áreas, datas, números de documentos).

### 3.2 Estrutura de Considerandos: Fato -> Artigo -> Cálculo
Cada item da lista de `considerandos` deve seguir uma progressão lógica tripartite:
1. **Fato:** O que foi constatado (ex: "foi verificado o acréscimo de 85,00m²").
2. **Artigo:** Qual norma enquadra o fato (ex: "nos termos do Art. 79 da Lei 1.544/86").
3. **Cálculo:** O resultado matemático da aplicação da norma (ex: "resultando em multa de R$ 3.482,28").

### 3.3 Precedência do Decreto 4.149/2019
Em todos os campos de `fundamentacao_legal`, o **Decreto Municipal nº 4.149/2019** deve ser o primeiro item citado, por regulamentar o rito administrativo de todos os processos de aprovação e regularização na SMOSU.

### 3.4 Formatação Markdown
Os campos narrativos devem utilizar Markdown básico para garantir a legibilidade no documento final (.docx):
- **Negrito:** Usar `**texto**` para dados críticos (áreas, valores, nomes, artigos).
- **Itálico:** Usar `*texto*` para termos em latim ou citações específicas.
- **Quebras de linha:** Usar `\n` para separar parágrafos dentro de strings JSON.
