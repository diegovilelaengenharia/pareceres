# PLANO — Fase 11: Qualidade e Inteligência dos Pareceres
## 11-01: Wizard Interativo + Reescrita dos Prompts

**Objetivo**: Transformar o GEM em um assistente interativo que guia o engenheiro por uma seleção de categoria, extrai dados dos PDFs com precisão, propõe a lista de documentos a emitir e só gera o JSON após confirmação explícita.

**Contexto**: Leia o CONTEXT.md e o 05_MAPA_INTELIGENCIA.md antes de executar.

---

## ARQUITETURA DO NOVO FLUXO

### Problema atual
A IA recebe 36 tipos possíveis, tenta adivinhar qual é, frequentemente erra → JSON com campos errados → parecer incompleto.

### Solução: Wizard de 3 Camadas

```
CAMADA 1 — PRÉ-SELEÇÃO (10 segundos do engenheiro)
  Engineer seleciona a categoria → IA já sabe 50% do contexto

CAMADA 2 — ANÁLISE DIRIGIDA (IA trabalha com foco)
  IA extrai os campos específicos da categoria → calcula índices → detecta exceções

CAMADA 3 — PROPOSTA + CONFIRMAÇÃO (decisão do engenheiro)
  IA lista os documentos a emitir → engenheiro confirma/ajusta → IA gera JSON
```

## PERGUNTAS DE DISAMBIGUAÇÃO POR CATEGORIA

### Categoria 1 — Obra Nova
```
Após seleção [1]:
"Qual o uso da edificação?
  [A] Residencial unifamiliar (padrão)
  [B] Residencial MCMV / CEF / programa habitacional
  [C] Comercial (loja, escritório, serviços)
  [D] Galpão / depósito / industrial"
```

### Categoria 2 — Regularização
Sem pergunta adicional — extração direta de todos os campos da regularização.

```
Após seleção [2], a IA vai direto:
"Analisando... extraindo dados da regularização."
```

### Categoria 3 — Habite-se
```
Após seleção [3]:
"O fiscal identificou alguma infração?
  [A] Não — obra conforme o alvará
  [B] Sim — quebra de permeabilidade, TO, ou outra infração"
```

### Categoria 4 — Reforma / Ampliação / Demolição
```
Após seleção [4]:
"O que inclui este processo?
  [A] Só reforma interna (sem demolição nem ampliação)
  [B] Só ampliação (acréscimo de área)
  [C] Demolição parcial + ampliação (+ possível reforma)
  [D] Só demolição"
```

### Categorias 5–10
Sem pergunta adicional — categoria já determina o tipo com precisão suficiente.

## FLUXO COMPLETO PÓS-SELEÇÃO

### ETAPA A — Extração Dirigida
Após a seleção, a IA executa o checklist específico da categoria:

**Para categorias 1-4 (pareceres técnicos):**
```
Extraindo obrigatoriamente:
✓ numero_processo + data_processo
✓ requerente (MAIÚSCULAS)
✓ inscricao_municipal
✓ logradouro + bairro
✓ area_terreno + area_total_construida
✓ zona_uso (inferida do bairro + tabela de zoneamento)
✓ art_rrt + profissional_nome + tipo_RT (ART/RRT/TRT)
✓ agentes_fiscais (nome + matrícula)
✓ situacao_obra ("não iniciada" / "concluída e habitável" / etc.)

Calculando:
✓ TO = área_projeção / área_terreno × 100
✓ CA = área_total_construida / área_terreno
✓ TP = área_permeável / área_terreno × 100
✓ Verificação contra limites da zona

Para regularização, adicionalmente:
✓ area_ja_averbada (se consta AV-X na matrícula)
✓ area_acrescimo = total − já_averbada
✓ area_decadencia (se planta cadastral >5 anos)
✓ Cálculo de multa Art. 79 + Art. 39 (se aplicável)
```
